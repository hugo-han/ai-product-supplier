第1页 — 背景与目标

标题：AIBP 协同销售的 Agent 方案 — 背景与目标

要点：
1. 我是谁：研发中心的 AIBP，协同销售通过 Coze 开发 Agent，提高销售效率
2. 本次任务：允许销售通过“关键字或图片”查询各电商平台的商品信息、价格与主图链接
3. 输出目标：标准化商品列表（商品名 / 价格 / 主图 URL / 平台 / 置信度），降低检索时间与操作门槛

讲稿要点（30–45 秒）：
- 我们希望把销售在多平台检索商品的耗时工作，交给智能 Agent 完成，核心输入是关键词或图片，输出为标准化商品卡片，便于报价与谈单。
- 强调用户诉求：一线销售需要简单、可靠且可追溯的检索结果。

第2页 — 规划与演示

标题：规划与使用演示（关键词 / 图片检索）

要点：
1. 解决思路：Coze Agent 接受关键词或图片 → 调度查询（优先 API / 备用爬虫 / 第三方数据）→ 聚合与标准化 → 返回 TopN 结果
2. 核心能力：输入（关键词/图片）｜输出（商品名、价格、主图、平台、链接、置信度）｜缓存与去重
3. 演示脚本：
   - 关键字检索：输入“运动手表” → 返回 Top5（含价格与主图链接）→ 点击查看详情
   - 图片检索：上传主图 → 返回相似或同款候选（显示匹配度与来源）

讲稿要点（45–60 秒）：
- 简述系统流程与落地演示的步骤，说明缓存/聚合如何减少重复请求与提升稳定性；演示时说明失败兜底（未查到则提示人工渠道）。

插画 Prompt（中文）：
手绘风，左侧为搜索输入与上传图片的简洁 UI mock（手写风搜索框、上传框），右侧为结果卡片列（每张卡含缩略图、商品名、价格、平台手绘标识但不含真实 logo），用手绘箭头标注“点击查看”，色调柔和、纸纹质感，2048x2048。

第3页 — 卡点与我们需要的支持

标题：卡点与待决支持请求

卡点（编号）：
1. 场景与可用性：销售需求偏向“工具/平台使用”，对业务人员仍有一定技术/流程门槛 → 采纳率风险
2. 反爬与稳定性：关键词/图片查询在技术上类似爬虫，主流电商反爬策略强，易出现抓取失败或 IP 被封 → 查询覆盖/可用率受影响
3. 登录与账号风险：获取价格/主图常需账号登录（定制化、账号管理复杂且存在封号风险）

简短缓解建议（每项 1 行）：
- 降低门槛：提供模板化查询、运营/BD 协助
- 稳定性：优先争取官方 API / 第三方数据合作；若抓取需建设 IP/代理/限速和缓存
- 账号问题：走授权/合作方式（API key / OAuth / 第三方数据），避免使用大量人工账号

对高管的请求（明确三项）：
1. 批准与主要电商平台/数据方的 API/数据合作预算与法律支持
2. 授权运维/安全团队提供抓取所需基础设施或明确禁抓取策略
3. 决策：是否允许采购第三方数据服务（并承担合规/费用）

讲稿要点（45–60 秒）：
- 逐条说明每个卡点对可用率、成本与合规的影响，并强调若能获得 API 合作，项目风险将显著下降；最后明确请求高层做出上述决策以保证能按期交付 MVP。

插画 Prompt（中文）：
手绘漫画式三格，每格表现一个卡点：1）销售人员面带困惑与工具复杂的场景；2）爬虫被拦截示意（拦截盾牌、断开的网线）；3）账号被封的夸张表情与锁链；每格配手写式注释，整体风格温和但表达风险，2048x2048。
